# Task 3: The System Agent — Implementation Plan

## 1. Цель
Добавить инструмент query_api для общения с бэкендом и научить агента отвечать на вопросы о системе и данных.

## 2. Новый инструмент: query_api
- **Назначение:** отправлять запросы к бэкенду
- **Параметры:** method (GET/POST), path, body (опционально)
- **Аутентификация:** LMS_API_KEY из .env.docker.secret
- **Безопасность:** ключ берётся из переменных окружения, не хардкодится

## 3. Обновление системного промпта
Агент должен понимать, когда использовать:
- list_files/read_file — для вопросов по документации
- query_api — для вопросов о данных (сколько предметов, статус)

## 4. Переменные окружения (ВАЖНО!)
- LLM_API_KEY, LLM_API_BASE, LLM_MODEL — из .env.agent.secret
- LMS_API_KEY — из .env.docker.secret (для query_api)
- AGENT_API_BASE_URL — базовый URL бэкенда (по умолчанию http://localhost:42002)

## 5. Тестирование
- 2 новых теста: один на read_file, один на query_api
- Запуск run_eval.py для проверки всех 10 вопросов

## 6. Ожидаемые проблемы
- Неправильный ключ LMS_API_KEY
- Неверный базовый URL
- LLM путает, когда какой инструмент использовать
